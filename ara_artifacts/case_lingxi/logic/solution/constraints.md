# Constraints

## Data And Market Constraints

1. Some global-market datasets use static large-cap universes and may include survivorship bias.
2. Crypto trades 24/7, but the current protocol evaluates daily H5 portfolios.
3. Industry/size neutralization is more meaningful for equities than for crypto.
4. Yahoo Finance sourced global panels do not fully replace professional point-in-time institutional data.

## Method Constraints

1. The structured market tags are deterministic proxies, not live LLM debate outputs.
2. The RL router is a small tabular baseline, not a claim about all possible RL methods.
3. The iTransformer/VarAttn and other SOTA proxies are lightweight approximations unless official implementations are explicitly imported.
4. Current dynamic routers are research-only.

## Claim Constraints

1. Production claims are limited to the current benchmark evidence.
2. Future OOS evidence can supersede the current static menu.
3. ARA Level 1 validation checks structure, not full semantic rigor.
